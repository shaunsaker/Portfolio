Triple M Web App

Triple M approached me to create a website for their Music Trivia and DJ business. I had loads of fun creating it because it turned out to be more a web app than anything else. The following unique features were added:

  • 1. The app is very simple and concise. In order to keep a user's attention (and add a fun element), I created a simple Music Trivia flash card game.

  • 2. Mike and Sharon didn't want to manually update their events on their site every time they created a new event on their Facebook page. And I didn't want to include a Facebook widget (which would have been easier but it would not have looked consistent with the design of the rest of the page). Instead I made use of Facebook's Graph API to dynamically pull in these events from Triple M's Facebook page and generate my own custom Event cards.

React Dungeon Crawler App

I created this app as part of the Free Code Camp React JS course.

The following user stories were fulfilled:

  • 1. I have health, a level, and a weapon. I can pick up a better weapon. I can pick up health items.

  • 2. I can move throughout a map, discovering items.

  • 3. I can move anywhere within the map's boundaries, but I can't move through an enemy until I've beaten it.

  • 4. Much of the map is hidden. When I take a step, all spaces that are within a certain number of spaces from me are revealed.

  • 5. When I beat an enemy, the enemy goes away and I get XP, which eventually increases my level.

  • 6. When I fight an enemy, we take turns damaging each other until one of us loses. I do damage based off of my level and my weapon. The enemy does damage based off of its level. Damage is somewhat random within a range.

  • 7. When I find and beat the boss, I win.

  • 8. The game should be challenging, but theoretically winnable.

Simon Says App

I created this app as part of the Free Code Camp Front End Development Certification.

The following user stories were fulfilled:

  • 1. I am presented with a random series of button presses.

  • 2. Each time I input a series of button presses correctly, I see the same series of button presses but with an additional step.

  • 3. I hear a sound that corresponds to each button both when the series of button presses plays, and when I personally press a button.

  • 4. If I press the wrong button, I am notified that I have done so, and that series of button presses starts again to remind me of the pattern so I can try again.

  • 5. I can see how many steps are in the current series of button presses.

  • 6. If I want to restart, I can hit a button to do so, and the game will return to a single step.

  • 7. I can play in strict mode where if I get a button press wrong, it notifies me that I have done so, and the game restarts at a new random series of button presses.

  • 8. I can win the game by getting a series of 20 steps correct. I am notified of my victory, then the game starts over.
